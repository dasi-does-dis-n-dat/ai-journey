What I Actually Learned Today

GitHub is basically a place where developers store their code and projects. Think of it like Google Drive, but for code, and with a slightly more intimidating interface designed to make normal people feel like they've accidentally walked into NASA.

Here's what all the confusing words actually mean:

Repository A folder for your project. Not a bank. I asked.
Commit Saving your work with a little note. Like a diary entry but for code.
Branch A copy of your project to experiment in without breaking the original. Very useful when you have no idea what you're doing, which is always.
Pull Request Asking to merge your changes back into the main project. Sounds aggressive. Is not.
README The front page of your repo. The first thing anyone sees. Mine took three attempts to upload correctly.
Repo vs Project A repo is where your files live. A project is a planning board on top. I only need repos right now. Good. One less thing.
