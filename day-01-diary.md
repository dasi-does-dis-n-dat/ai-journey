## What I Actually Learned Today

**GitHub** is basically a place where developers store their code and projects. Think of it like Google Drive, but for code, and with a slightly more intimidating interface designed to make normal people feel like they've accidentally walked into NASA.

Here's what all the words mean that I had to ask about:

- **Repository (repo)** — a folder for your project. Not a bank. I asked.
- **Commit** — saving your work with a little note explaining what you did. Like a diary entry but for code.
- **Branch** — a copy of your project where you can experiment without breaking the original. Very useful when you have no idea what you're doing, which is always.
- **Pull Request** — asking to merge your changes back into the main project. Sounds aggressive. Is not.
- **README** — the front page of your repo. The first thing anyone sees. Mine took three attempts to get onto GitHub correctly.
- **Repository vs Project** — a repo is where your files live. A project is a planning board on top of it. I only need repos right now. Good, because that's one less thing to be confused about.

## Silly Questions I Asked (So You Don't Have To)

- *"What is GitHub?"* — asked within 30 seconds of starting
- *"Will people know it's me?"* — yes, if you use your actual name. I did not use my actual name.
- *"Can I change my name on GitHub?"* — yes. Do this before you tell anyone your username.
- *"Do I create a new repository or just upload files?"* — create the repo first, THEN upload. Shelf first, book second. 🪵📖
- *"Where is the README file we made?"* — turns out "we" was doing a lot of heavy lifting. Claude made it. I downloaded it. Team effort.
